Desenvolvimento Actual

Hoje em dia o desenvolvimento multiplataforma é ineficiente e complicado.

Existe uma grande duplicação do trabalho que desenvolvemos.

A nova funcionalidade que acabámos de desenvolver nesta plataforma tem de ser portada para as restantes.

O bug reportado nesta plataforma tem de ser agora corrigido em todas as outras, de forma a garantir que todas as plataformas estão no mesmo ponto de desenvolvimento.

Existe assim uma sincronização manual de implementações, correcções e desenvolvimentos para todas as plataformas.

Tornando assim este modo de desenvolvimento pouco agile e bastante dispendioso.

Why use MORE

Mas este modelo de desenvolvimento pode ser melhorado.

O desenvolvimento de soluções, sejam elas enterprise ou não, deve focar-se na qualidade e eficiência da mesma.

Esse deve ser o principal objectivo.

O Foco no produto final.

O período de desenvolvimento deve ser optimizado para reflectir as necessidades dos nossos clientes.

Assim, o foco enquanto developers deve ser na implementação lógica da nossa solução.

O foco deve recair sobre questões como:

  • qual a melhor forma de recolher o input dos utilizadores ou,
  • como apresentar informação relevante de forma simples e estruturada.

E para que o foco seja esse, não podem existir preocupações com:

  • implementações da camada de dados ou,
  • a camada de comunicações ou
  • a gestão de múltiplas sessões num servidor

Estes são aspectos base de um sistema. São configurações quase estáticas. Não mudamos o motor de base de dados de um projecto a cada release.

Mas então, como melhorar este modelo de desenvolvimento?

Certamente não é recorrendo a plataformas de geração de código, automáticas e com pouco espaço de costumização nativa. Além disso são ineficientes e têm um aspecto esquisito.

Nós acreditamos que no centro de qualquer desenvolvimento está o Modelo de Negócio que queremos implementar. A lógica de processos que torna possível a actividade dos nossos clientes. E isso deve ser abstraído de qualquer especificidade tecnológica e de qualquer plataforma. Deve ser desenvolvido no seu formato mais simples - de alto nível.

How MORE Works

A plataforma de desenvolvimento do MORE está montada de forma a sustentar este modelo.

No centro, temos o MORE Server que garante a sintonia entre todos os componentes.

O MORE Designer é a nossa aplicação para desenvolver toda a lógica de negócio da nossa solução e para desenharmos todas as interfaces, com um elevado nível de detalhe que permite virtualmente desenvolver qualquer UI.

A Biblioteca MORE presente em cada plataforma (iOS, Android, Windows e JavaScript) garante que o nosso modelo lógico será interpretado de igual forma independentemente da plataforma que o está a receber. A Biblioteca do MORE permite ainda pontos de extensibilidade, para que os nossos parceiros consigam desenvolver funcionalidades especificas para cada cliente. Estes pontos de extensibilidade existem tanto ao nível das aplicações em si como do lado do MORE Server - para tarefas mais complexas.

O MORE permite ainda que as soluções desenvolvidas possam ser desenhadas numa lógica online, offline ou ambas - SMART.

Em termos de integração com serviços externos, é o MORE Connector HUB que está encarregue de as assegurar. Com grande facilidade é possível ligar serviços como o Google Analytics, CRM, SAP ou uma integração custom às soluções desenvolvidas com o MORE.

What MORE is

Detalhando então um pouco mais, comecemos pelo MORE Designer.

Na coluna mais à esquerda temos o Solution Explorer.

Como o nome indica é onde está toda a estrutura da nossa solução.

Uma representação semelhante a um sistema de ficheiros. É responsável por organizar os elementos base da nossa solução - módulos, entidades e processos.

Na coluna mais à direita temos a Properties Table onde estão todas as properties dos objectos selecionados. As properties funcionam como modificadores em relação aos objectos a que são adicionadas.

Na coluna do lado esquerdo da Properties table temos o Form Configurator. Onde conseguimos facilmente desenhar e visualizar a hierarquia de fields de um determinado ecran.

Por fim, ao centro, tempos o Diagram Editor. Aqui é onde está representado o workflow que definimos para cada execução de processos. É onde estão representados os processos e flows da nossa lógica de modelo.

O MORE Designer está disponível como aplicação em Mac e Windows e temos ainda uma versão para o Browser desenvolvida em JavaScript.

MORE Features

Olhando agora para a componente mais técnica de uma implementação MORE.

Essa implementação pode ser feita de várias formas. Cloud based, Dockers ou on premises. Em termos de motores de base de dados suportamos, por exemplo: MS SQL, ORACLE, MySQL e AppEngine.

A título de exemplo, esta é uma implementação típica da Infraestrutura MORE num modelo on premises.

Onde temos uma Gateway exposta para a internet, capaz de encaminhar os pedidos para o servidor (numa infraestrutura segura) com ligações para o motor de BD.

Do ponto de vista de segurança o MORE suporta as seguintes normas de segurança:

  • Certificate Pinning
  • Obfuscated binaries (standard and custom tools)
  • Anti-Decompiling
  • Anti-Disassembling
  • Anti-Tampering

O Workflow de desenvolvimento em MORE é diferente do tradicional.

Com o MORE, depois de termos toda a lógica de negócio desenvolvida fazemos um export desse modelo gerando assim um script de actualização de modelo. Esse script é depois executado nos ambientes que desejamos actualizar. Tornando assim a tarefa de deploy muito mais simples e praticamente instantânea.

Ficamos assim livres de tempos de espera de aprovação nas Stores de Aplicações.

Podemos ainda fazer um Snapshot dos desenvolvimentos para backup.

Mudando assim o paradigma de desenvolvimento conseguimos abstrair a lógica de execução da nossa solução das implementações técnicas. O MORE baseia-se neste conceito MODEL ONCE RUN EVERYWHERE.
